📦 Phase 1 · ~10 minutes
Set up Supabase (Database + Auth)
Supabase is your free backend — it stores jobs, users, applications, notifications.
1
Create a free Supabase account
Go to the Supabase website and sign up
🌐 https://supabase.com → click "Start your project"
What to do
→Click "Sign Up" and create an account with your email or GitHub
→Once logged in, click "New project"
→Name it workdey, choose a region close to Cameroon (EU West or US East), set a database password → Save it somewhere!
→Wait ~2 minutes for the project to be created
2
Copy your API keys
You need 2 values: Project URL and anon/public key
Navigation path
→In your project dashboard, click ⚙️ Project Settings (bottom left sidebar)
→Click "API" in the settings menu
→You'll see two values — copy them both:
Values to copy from Supabase → Settings → API
// 1. Project URL — looks like this:
SUPABASE_URL = "https://abcdefghijklm.supabase.co"

// 2. Project API keys → anon / public — looks like this:
SUPABASE_ANON = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi..."
// (it's a very long string — copy the whole thing)
ℹ️
Which key to use? Use the anon / public key, NOT the secret key. The anon key is safe to put in your frontend app.
3
Create the database tables
Copy and paste this SQL into Supabase's SQL Editor — one click creates everything
Navigation path
→In the left sidebar, click 🗄 SQL Editor
→Click "New query"
→Paste ALL the SQL below, then click "Run" (▶)
🗃
workdey_tables.sql — paste this entire block into SQL Editor
-- ═══════════════════════════════════════
-- WORKDEY — Database Tables
-- Paste this into Supabase SQL Editor → Run
-- ═══════════════════════════════════════

-- 1. USER PROFILES
create table profiles (
  id          uuid references auth.users primary key,
  name        text,
  email       text,
  photo_url   text,
  role        text default 'seeker',   -- 'seeker' or 'recruiter'
  lang        text default 'en',
  phone       text,
  location    text,
  company     text,
  skills      text[],
  wa_number   text,
  bio         text,
  fcm_token   text,
  rating      numeric default 0,
  review_count int default 0,
  verified    boolean default false,
  created_at  timestamptz default now()
);
alter table profiles enable row level security;
create policy "Users manage own profile"
  on profiles for all using (auth.uid() = id);

-- 2. JOBS
create table jobs (
  id          uuid primary key default gen_random_uuid(),
  poster_id   uuid references profiles(id),
  title       text not null,
  company     text,
  location    text,
  type        text,    -- 'full-time' | 'part-time' | 'gig' | 'contract'
  industry    text,
  salary_min  int,
  salary_max  int,
  description text,
  skills      text[],
  wa_number   text,
  cv_required boolean default false,
  emoji       text,
  bg_color    text,
  experience  text,
  is_active   boolean default true,
  created_at  timestamptz default now()
);
alter table jobs enable row level security;
create policy "Anyone reads active jobs"
  on jobs for select using (is_active = true);
create policy "Recruiters manage own jobs"
  on jobs for all using (auth.uid() = poster_id);

-- 3. APPLICATIONS
create table applications (
  id          uuid primary key default gen_random_uuid(),
  job_id      uuid references jobs(id) on delete cascade,
  seeker_id   uuid references profiles(id),
  status      text default 'pending',  -- pending | viewed | accepted | rejected
  created_at  timestamptz default now(),
  unique(job_id, seeker_id)             -- one application per job per person
);
alter table applications enable row level security;
create policy "Seekers manage own applications"
  on applications for all using (auth.uid() = seeker_id);
create policy "Recruiters see apps on their jobs"
  on applications for select using (
    exists (
      select 1 from jobs
      where jobs.id = job_id
      and jobs.poster_id = auth.uid()
    )
  );

-- 4. NOTIFICATIONS
create table notifications (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid references profiles(id),
  type        text,
  title       text,
  body        text,
  icon        text default '🔔',
  color       text default 'var(--d3)',
  is_read     boolean default false,
  created_at  timestamptz default now()
);
alter table notifications enable row level security;
create policy "Users manage own notifications"
  on notifications for all using (auth.uid() = user_id);

-- 5. Enable Realtime on jobs and notifications
alter publication supabase_realtime add table jobs;
alter publication supabase_realtime add table notifications;

-- Done! You should see 4 green checkmarks above.
✅
Success looks like: You'll see "Success. No rows returned" in green. Your 4 tables now appear under Table Editor in the sidebar.
🔐 Phase 2 · ~10 minutes
Enable Google Sign-In
Two steps: create credentials in Google Cloud, then enable in Supabase. You need both.
4
Create a Google Cloud project & OAuth credentials
This gives you the Client ID and Secret that Google uses to identify your app
🌐 console.cloud.google.com
Step by step
①Sign in with your Google account
②Click "Select a project" at the top → "New Project" → Name it WorkDey → Create
③In the left menu: APIs & Services → OAuth consent screen
④Choose "External" → Click Create
⑤Fill in: App name = WorkDey, User support email = your email → click Save and Continue through all steps
⑥Now go to APIs & Services → Credentials
⑦Click "+ Create Credentials" → "OAuth client ID"
⑧Application type: "Web application" → Name: WorkDey Web
⑨Under "Authorized redirect URIs", click + Add URI and paste this exactly:
Authorized redirect URI — paste this in Google Cloud
https://YOUR-PROJECT-REF.supabase.co/auth/v1/callback

// Replace YOUR-PROJECT-REF with your actual Supabase project ref
// Find it in Supabase → Settings → General → Reference ID
// Example: https://abcdefghijklm.supabase.co/auth/v1/callback
After clicking Create
→A popup appears with Your Client ID and Your Client Secret
→Copy both values — you need them in the next step
5
Enable Google provider in Supabase
Paste the Google credentials into Supabase
Navigation: back in your Supabase dashboard
→Left sidebar: click 🔐 Authentication
→Click "Providers"
→Find Google in the list → click it to expand
→Toggle "Enable Sign in with Google" → ON
→Paste your Google Client ID and Google Client Secret from the previous step
→Click Save
✅
Google Sign-In is now active. Users can sign in to WorkDey with their Google account.
🔔 Phase 3 · ~10 minutes
Set up Push Notifications (FCM)
Firebase Cloud Messaging sends "New job near you" alerts to users' phones even when the app is closed.
6
Create a Firebase project
Firebase is free — same Google account works
🌐 console.firebase.google.com
Step by step
①Click "Add project" → Name it WorkDey
②Disable Google Analytics (not needed) → Click "Create project"
③Wait ~30 seconds → click "Continue"
7
Get your VAPID key for web push
This key lets your PWA receive push notifications
In your Firebase project
→Click the ⚙️ gear icon next to "Project Overview" → Project settings
→Click the "Cloud Messaging" tab at the top
→Scroll down to "Web configuration"
→Under "Web Push certificates", click "Generate key pair"
→A long key appears — copy it (this is your VAPID key)
ℹ️
Also copy these from Project Settings → General → Your apps: Add a web app (click </>), copy the firebaseConfig object — you'll need it for the service worker file.
8
Create the Firebase Service Worker file
This file must exist at the root of your website for push notifications to work
Create a file called: firebase-messaging-sw.js
→Place it in the same folder as your workdey.html file
→Or if hosting on a web server, place it at the root (e.g. yoursite.com/firebase-messaging-sw.js)
📄
firebase-messaging-sw.js — create this file
// firebase-messaging-sw.js
// Place this file at the root of your website

importScripts('https://www.gstatic.com/firebasejs/10.12.0/firebase-app-compat.js');
importScripts('https://www.gstatic.com/firebasejs/10.12.0/firebase-messaging-compat.js');

firebase.initializeApp({
  apiKey:            "YOUR_API_KEY",           // from Firebase Project Settings
  authDomain:        "YOUR_PROJECT.firebaseapp.com",
  projectId:         "YOUR_PROJECT_ID",
  storageBucket:     "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId:             "YOUR_APP_ID"
});

const messaging = firebase.messaging();

// Handle background notifications (app is closed)
messaging.onBackgroundMessage((payload) => {
  console.log('[WorkDey SW] Background message:', payload);
  self.registration.showNotification(payload.notification.title, {
    body:  payload.notification.body,
    icon:  '/icon-192.png',
    badge: '/icon-72.png',
    data:  payload.data
  });
});
⚠️
Replace the placeholder values with your real Firebase config. Go to Firebase Console → Project Settings → General → Your apps → Web app → click the icon → copy the config values.

🚀 Phase 5 · 5 minutes
Host your app online (free)
For Google OAuth and push notifications to work, your app must be on a real URL — not just opened as a local file.
10
Option A — Netlify Drop (easiest, 30 seconds)
Drag your folder onto Netlify and get a live URL instantly
🌐 app.netlify.com/drop
How to deploy
①Create a folder on your computer called workdey
②Put workdey.html inside it — rename it to index.html
③Also put firebase-messaging-sw.js in the same folder
④Go to app.netlify.com/drop → drag the entire folder onto the page
⑤Netlify gives you a URL like random-name-123.netlify.app — that's your live app!
⚠️
Important — update Google redirect URI: After getting your Netlify URL, go back to Google Cloud Console → Credentials → your OAuth client → add https://your-app.netlify.app to "Authorized JavaScript origins" and update the redirect URI.

🛠 Common Issues
!
"Google sign-in popup closes immediately"
Fix: The redirect URI isn't configured correctly
Go to Google Cloud → Credentials → your OAuth client → add your exact site URL to Authorized JavaScript origins (e.g. https://workdey.netlify.app) AND the Supabase callback URL to Authorized redirect URIs.
!
"Jobs not loading / database error"
Fix: Check your Supabase URL and anon key
Open your browser DevTools (F12) → Console tab. If you see a Supabase error, verify the URL starts with https:// and the anon key is the full long string. Also check that the SQL ran successfully by going to Supabase → Table Editor and confirming 4 tables exist.
!
"Push notifications not working"
Fix: Service worker must be on HTTPS and same origin
firebase-messaging-sw.js must be at the root of your website (same URL as your app). It won't work if you open the HTML file locally (file://). Make sure you're on Netlify/Vercel with HTTPS.
